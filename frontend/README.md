# IHaveWatched — Frontend

Next.js 16 (App Router, Turbopack) frontend for the IHaveWatched collaborative media-tracking app. See the [root README](../README.md) for full project overview.

## Quick start

```bash
npm install
cp .env.local.example .env.local   # fill in your SpacetimeDB + TMDB values
npm run dev
```

## Key directories

| Directory | Purpose |
|---|---|
| `app/` | Pages and API route handlers (App Router) |
| `components/canvas/` | react-konva node canvas, watch chips, drill-down drawer |
| `components/board/` | TMDB search and import UI |
| `components/ui/` | Shared UI primitives (Button, Input, Modal, etc.) |
| `lib/db/` | Identity token management, TMDB import orchestration |
| `lib/canvas/` | Zoom-level layout engine, node dimensions |
| `lib/hooks/` | Camera controls, async Konva image loading |
| `src/module_bindings/` | Auto-generated by `spacetime generate` — **do not edit** |

## Auth

Anonymous SpacetimeDB identities. Enter a display name on first visit; the SDK issues an identity + token stored in `localStorage`. No OAuth, no passwords.

## Build

```bash
npm run build   # must pass with no type errors
```

All board pages use `export const dynamic = 'force-dynamic'` to prevent Next.js prerender errors with SpacetimeDB hooks. The `providers.tsx` shell defers SpacetimeDBProvider to client mount to avoid SSR crashes.
