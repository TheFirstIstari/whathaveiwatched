# Deploy steps

## 1. SpacetimeDB module (run on home server)

```bash
# SSH into home server
ssh frobinson@fedora-1  # or 100.82.166.71

# Pull latest module code
cd /opt/whathaveiwatched-module  # or wherever the module lives
git pull origin main

# Build + publish
spacetime publish whathaveiwatched \
  --server http://127.0.0.1:3000 \
  --yes \
  --bin-path target/wasm32-unknown-unknown/release/ihavewatched_module.wasm
```

If `spacetime publish` gives a 403 (wrong owner), reset the server data dir first:

```bash
spacetime server clear --yes --data-dir /var/lib/spacetime
mkdir -p /var/lib/spacetime/{databases,snapshots,logs}
# Then republish
```

## 2. Frontend (automatic — already triggered by push to main)

The GitHub Actions deploy workflow handles this:
- Joins Tailnet via secret
- rsyncs `frontend/` to home server
- Runs `npm ci && npm run build`
- Restarts pm2 process

You can monitor it at: https://github.com/TheFirstIstari/whathaveiwatched/actions

## 3. Verify

```bash
# After deploy completes, smoke test
curl https://watched.tweak.wiki/api/health
# Check the board page loads with new canvas interactions
```

## What's new

- **Drag-to-reorder**: Drag nodes on the canvas to reorder them (owner only)
- **Minimap**: Bottom-right corner overview with viewport indicator
- **Multi-select**: Shift+click, Ctrl+A, bulk operations via right-click menu
- **Keyboard shortcuts**: Escape (clear/dismiss), Delete (remove), Arrow keys (navigate), Enter/Space (toggle watch)
- **Zoom controls**: Buttons + scale indicator, pinch-to-zoom on mobile
- **Connection lines**: Arrows between chronologically adjacent items in a lane
- **Performance**: Indexed SpacetimeDB queries instead of full table scans
